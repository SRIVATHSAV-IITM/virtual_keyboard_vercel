# Project Architecture and Flow

## 1. Project Overview

This project is a multi-application suite managed by a central main menu. The suite is designed to be controlled primarily through gaze tracking, providing a hands-free user experience. It includes a variety of applications, such as games and utilities, each launched from the main menu.

The core technologies used are Python for the backend logic, Eel for creating the web-based UI, Pygame and OpenCV for the application interfaces, and MediaPipe for the gaze tracking functionality.

## 2. Core Components

- **`main.py`**: This is the main entry point of the application. It initializes the main menu, manages the gaze tracker, and handles the launching and monitoring of all sub-applications.

- **`gaze_tracker.py`**: A crucial module that uses `opencv-python` and `mediapipe` to track the user's gaze. It captures video from the camera, detects facial landmarks, and maps the user's iris position to screen coordinates. This module is designed to be robust, automatically detecting and connecting to an available camera.

- **`web/`**: This directory contains the source code for the main menu's user interface, including `main.html`, `main.js`, and `style.css`.

- **`logs/`**: All log files generated by the main application and sub-applications are stored in this directory.

- **Sub-application Directories**:
    - `tic_tac_toe/`
    - `snake_game/`
    - `virtual_keyboard/`
    - `activities_center/`

## 3. Application Flow

The application follows a clear and consistent flow:

1.  **Initialization**: The user runs `main.py`, which initializes Pygame, gets the screen dimensions, and starts the main menu UI using Eel.

2.  **Gaze Tracker Activation**: A dedicated thread is started to run the gaze tracker, allowing the user to control the main menu with their gaze.

3.  **Sub-application Launch**: When the user selects an application from the main menu, the following steps occur:
    - The main application's gaze tracker is paused and its resources are released to make the camera available.
    - The selected sub-application is launched as a separate process using Python's `subprocess` module.

4.  **Sub-application Execution**: The sub-application runs independently. If it is a gaze-controlled application (like the Snake game or Virtual Keyboard), it initializes its own instance of the `GazeTracker`.

5.  **Return to Main Menu**: When the user closes a sub-application, the following happens:
    - The sub-application's process is terminated.
    - The main application detects that the sub-process has ended and automatically re-initializes the gaze tracker for the main menu.

## 4. Gaze Tracking and Camera Management

The camera is a critical resource that is carefully managed to prevent conflicts between the main application and sub-applications.

- **Main Gaze Tracker**: The `main.py` script maintains a primary `GazeTracker` instance that runs in a background thread. This allows for continuous gaze tracking while the user is interacting with the main menu.

- **Resource Release**: Before launching any sub-application, the main application calls the `release()` method on its `GazeTracker` instance. This closes the camera and frees it up for the sub-application to use.

- **Re-initialization**: The main application continuously monitors the active sub-process. Once the sub-process terminates, the main application re-initializes its `GazeTracker`, allowing the user to seamlessly switch back to controlling the main menu with their gaze.

- **Robust Camera Detection**: The `GazeTracker` is designed to be resilient. Upon initialization, it iterates through a range of camera indices (0 through 4) and connects to the first available camera it finds. This avoids issues where the default camera is not available or is being used by another application.

## 5. Sub-application Details

### Tic-Tac-Toe

- **Launch**: The game is launched via the `launch_tic_tac_toe()` function in `main.py`.
- **Technology**: It is an Eel-based application, with the game logic implemented in HTML, CSS, and JavaScript.
- **Control**: The game is controlled with a standard mouse and keyboard.
- **Lifecycle**: The game runs in a separate process and, when closed, the main application's gaze tracker is re-initialized.

### Snake Game

- **Launch**: The game is launched via the `launch_snake_game()` function in `main.py`.
- **Technology**: It is a Pygame application that uses OpenCV for rendering.
- **Control**: The game is controlled by the user's gaze, with the snake's movement determined by where the user is looking on the screen.

### Virtual Keyboard

- **Launch**: The keyboard is launched via the `launch_virtual_keyboard()` function in `main.py`.
- **Technology**: It is a Pygame application that uses OpenCV for the UI.
- **Control**: The keyboard is controlled by gaze, allowing the user to type by looking at keys on the screen.

### Activities Center

- **Launch**: The center is launched via the `launch_activities_center()` function in `main.py`.
- **Technology**: It is a Pygame-based application.
- **Control**: It is designed to be controlled by gaze, providing access to a variety of activities and exercises.
